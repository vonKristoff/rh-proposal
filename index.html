Interactive live streams proposal

multiview video stream

concept

The main live edited video content has x3 extra angles that the user can switch between at will.

technology

The HD video stream is comprised of all 4 feeds into one 2x2 grid that works as a single stream.

A web based video player will be able to facilitate the User Interaction.

The Video player can be deployed onto the RH website as a inline script. This can take a URL of the comprised feed, and so the app can have a reusable lifecycle.

development

I propose the project be broken down into a 3 week development phase.

  • week 1

Develop the web player to a working prototype.

  • week 2

Integrate the player as a drop-in app with a video feed. This will require working with the live media department - or whoever is in charge of streaming.

  • week 3

Platform Testing, Q&A, performance and security enhancements.

There would be some prerequisite days to discuss further the project, and its immediate limitations that on agreement would allow for a swift execution and delivery of the development.

hotspot content over live video stream

concept

Be able to stream a video feed, with HTML interactive content that can deployed in a live manner by a RH Operator at any point during the event. An example would be an infomation hotspot would appear over a performer or stage equiptment, that the user could click upon and find out infomation about them/it.

Data could easily be pre-prepared, and set to go on command. This could lead to some exciting capabilities of the app, by becoming a similar tool to that of a Lighting Desk, which can have all kinds of live/programmed controls.

technology

There would be x3 applications involved in making the system work.

  • The User-side video player that can receive live data.
  • A Server API instance (maybe on Amazon AWS) to provide live data instantly easily to over 10’000 connections
  • An internal production tool running through a web browser that a RH personal would control the timings and execution of the live feeds. This passes the data to the API which then redirects updates to any connections, clientside.

The smart thing about this method is that all processing of data would actually be done by the user’s device, ensuring a super high speed connection from the RH server. The App would supply data, and the user’s browser would render, allowing for all the normal implications of HD quality video.

development

Based upon delivery of the first project, it would follow the same path regarding 2 weeks of Testing and Q&A. Then as there are x3 applications that connect the master system together, a similar 3 phases would be required.

  • 2 weeks video player - with hotspot technology
  • 1 week Server API
  • 2 - 3 weeks Operator Tool

Estimated Total: 8 weeks
